What it handles

The ongoing work of running a home, organized in one place.

Memory you don’t have to manage

Once you tell MyCasaPro something, it is stored, structured, and easy to find later. No notes app sprawl. No re-explaining.

Messages and follow-ups

Identifies important emails and messages, tracks open threads, and helps you close loops instead of rereading inboxes.

Bills and deadlines

Keeps bills, due dates, and reminders visible. Flags problems early so you are not reacting late.

Home upkeep

Tracks service history, schedules seasonal work, and keeps records you need when something breaks.

Plans into steps

Turns vague plans into concrete next actions so progress does not depend on memory.

Review before action

Planned actions are checked for scope and permission before anything is scheduled or sent.

One system. No confusion.

Information is stored, plans are reviewed, and actions are executed separately. That separation keeps mistakes from compounding.

Install locally in minutes

MyCasaPro is open-source and runs locally. No signup required.

Step 1 — Get the code

Option A: Clone the repository

git clone https://github.com/Fuzzyslippers412/Mycasapro.git
cd Mycasapro

Option B: Download ZIP

Download the repository directly

Step 2 — Install dependencies

npm install

Step 3 — Configure environment

Copy the example environment file and adjust as needed.

cp .env.example .env

Step 4 — Run the app

npm run dev

The app will be available at:

http://localhost:3000

Step 5 — Open it on other devices

Once it is running on your computer, you can open it on iPads, phones, or other devices on the same Wi-Fi network.

If it does not load from another device, start the dev server with a network hostname: 

npm run dev -- --hostname 0.0.0.0

Then visit:

http://YOUR_COMPUTER_IP:3000
