What it handles

The ongoing work of running a home, organized in one place.

Memory you don't have to manage

Once you tell MyCasaPro something, it's stored, structured, and easy to find later. No notes app sprawl. No re-explaining.

Messages and follow-ups

Identifies important emails and messages, tracks open threads, and helps you close loops instead of rereading inboxes.

Bills and deadlines

Keeps bills, due dates, and reminders visible. Flags problems early so you're not reacting late.

Home upkeep

Tracks service history, schedules seasonal work, and keeps records you actually need when something breaks.

Plans into steps

Turns vague plans into concrete next actions so progress doesn't depend on memory.

One system. No confusion.

Information is stored, plans are reviewed, and actions are executed separately. That separation keeps mistakes from compounding.

How it works

Capture, store, review, act - without losing context.

Capture

You share information by chat, email, or connected tools.

Store

Everything is saved in a clear, searchable timeline.

Review

Planned actions are checked for scope and permission.

Act

Only approved actions are scheduled or sent.

Install

Runs locally. No signup required.

  1. Get the code

    git clone https://github.com/Fuzzyslippers412/Mycasapro.git
cd Mycasapro

  2. Run the setup wizard

    ./mycasa setup

  3. Start everything

    MYCASA_API_PORT=6709 ./start_all.sh

  4. Open the app

    http://localhost:3000

  5. Use it on other devices

    Run once, then open on the same Wi-Fi:

    MYCASA_PUBLIC_HOST=<your-lan-ip> MYCASA_BIND_HOST=0.0.0.0 ./start_all.sh

    http://YOUR_LAN_IP:3000
